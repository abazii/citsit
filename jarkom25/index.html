2. FLSM & VLSM (Subnetting Strategy)

Definisi Strategi:

  • FLSM (Fixed Length): Semua subnet memiliki ukuran mask yang sama. Mudah dikelola, boros IP.
  • VLSM (Variable Length): Subnet dikalokasikan berdasarkan jumlah host terbanyak ke terkecil. Sangat hemat IP.
Rumus Dasar:
• Jumlah IP = 2(32 - prefix)
• Host Usable = Total IP - 2 (NA & BA)

5. Routing Configuration (Static & Dynamic)

📍 STATIC ROUTE
# ip route [NA] [Mask] [Next-Hop]
Router(config)# ip route 192.168.20.0 255.255.255.0 10.10.10.2
⚡ RIP VERSION 2
Router(config)# router rip
Router(config-router)# version 2
Router(config-router)# no auto-summary
Router(config-router)# network 192.168.10.0
⚡ OSPF (SINGLE AREA)
# Gunakan Wildcard Mask (WC)
Router(config)# router ospf 1
Router(config-router)# network 192.168.10.0 0.0.0.255 area 0

6. Route Redistribution (Cross-Protocol)

🔁 OSPF → RIPv2
Router(config)# router rip
Router(config-router)# redistribute ospf 1 metric 1
// RIP wajib menyertakan nilai 'metric' (hop count) agar rute dapat dipahami
🔁 RIPv2 → OSPF
Router(config)# router ospf 1
Router(config-router)# redistribute rip subnets
// Gunakan keyword 'subnets' agar rute classless dapat diredistribusi
🔁 Static Route → OSPF / RIPv2
# OSPF: Router(config-router)# redistribute static subnets
# RIP: Router(config-router)# redistribute static
🔁 Default Route (0.0.0.0/0) Advertisement
Router(config)# ip route 0.0.0.0 0.0.0.0 10.10.10.2
Router(config)# router ospf 1
Router(config-router)# default-information originate

7. VLAN & Inter-VLAN Routing

🎛️ SWITCH CONFIG (ACCESS & TRUNK)
# 1. Deklarasi VLAN Database
Switch(config)# vlan 10 → name Staff
Switch(config)# vlan 20 → name Guest
# 2. Alokasikan Interface ke VLAN (Access)
Switch(config)# interface fa0/5
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10
# 3. Port Trunk (Penghubung Router/Switch)
Switch(config)# interface fa0/1
Switch(config-if)# switchport mode trunk
🛰️ ROUTER CONFIG (ROUTER-ON-A-STICK)
# 1. Aktifkan Interface Fisik
Router(config)# interface fa0/0
Router(config-if)# no shutdown → exit
# 2. Sub-Interface VLAN 10
Router(config)# interface fa0/0.10
Router(config-subif)# encapsulation dot1Q 10
Router(config-subif)# ip address 192.168.10.1 255.255.255.0
# 3. Sub-Interface VLAN 20
Router(config)# interface fa0/0.20
Router(config-subif)# encapsulation dot1Q 20
Router(config-subif)# ip address 192.168.20.1 255.255.255.0

8. Security Filtering with Access Control List (ACL)

🛡️ EXTENDED ACL (Studi Kasus Server)

Batasi hanya ruang admin (192.168.17.128/28) yang boleh akses Admin Server (192.168.17.166).

# 1. Izinkan R.Admin (Wildcard /28 = 0.0.0.15)
Router(config)# access-list 100 permit ip 192.168.17.128 0.0.0.15 host 192.168.17.166
# 2. Tolak akses host lain ke Server
Router(config)# access-list 100 deny ip any host 192.168.17.166
# 3. Izinkan trafik sisa
Router(config)# access-list 100 permit ip any any
🔗 PENERAPAN INTERFACE

Aturan: Extended ACL diletakkan sedekat mungkin dengan pengirim (source) pada interface arah Outbound.

Router(config)# interface fastEthernet 1/0
Router(config-if)# ip access-group 100 out
Router(config-if)# exit

📝 Simpan Config:
Router# copy running-config startup-config
